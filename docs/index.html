Finance Dashboard API

A REST API for financial record management with role-based access control. Built with Node.js, Express, and SQLite (sql.js). Supports JWT authentication, soft delete, pagination, and dashboard analytics.

Financial Records
GET /records List records with filters & pagination All roles ▼
Query params (all optional)
?type=income|expense ?category=Salary ?startDate=2024-01-01 ?endDate=2024-12-31 ?page=1 ?limit=20 (max: 100)
Response 200
{ "records": [ { "id": "uuid", "amount": 5000, "type": "income", "category": "Salary", "date": "2024-06-01", "notes": "June salary", "created_by": "user-uuid", "created_by_name": "Bob Analyst", "created_at": "2024-06-01T10:00:00" } ], "total": 42, "page": 1, "limit": 20, "pages": 3 }
GET /records/:id Get a single record by ID All roles ▼
Example
GET /api/records/3f2a8c1d-4b5e-...
POST /records Create a new record Analyst / Admin ▼
Request body
{ "amount": 5000, "type": "income", "category": "Salary", "date": "2024-06-01", "notes": "June salary" }
Validation rules
amount required · positive number type required · "income" or "expense" category required · max 100 chars date required · YYYY-MM-DD format notes optional · max 500 chars
Response 201
{ "record": { ... } }
PATCH /records/:id Partially update a record Analyst / Admin ▼
Request body (provide any fields to update)
{ "amount": 6000 } { "category": "Bonus", "notes": "Q2 bonus" } { "type": "income", "date": "2024-07-01" }
Response 200
{ "record": { ... updated record ... } }
DELETE /records/:id Soft delete a record Admin only ▼
Behaviour
Sets deleted_at timestamp — record is hidden from all queries but preserved in the database for audit purposes.
Response 200
{ "message": "Record deleted successfully" }
User Management
GET /users List all users with filters Admin only ▼
Query params (optional)
?status=active|inactive ?role=viewer|analyst|admin ?page=1&limit=20
GET /users/:id Get a single user Admin only ▼
Response 200
{ "user": { "id": "...", "name": "...", "email": "...", "role": "...", "status": "active" } }
POST /users Create a new user Admin only ▼
Request body
{ "name": "John Doe", "email": "john@company.com", "password": "SecurePass1!", "role": "viewer" }
Roles
viewer read-only dashboard access analyst read + create/update records + analytics admin full access including user management
Response 201
{ "user": { "id": "...", "name": "John Doe", "role": "viewer" } }
PATCH /users/:id Update name, role, or status Admin only ▼
Updatable fields
{ "name": "New Name" } { "role": "analyst" } { "status": "inactive" }
Guard rules
Admin cannot deactivate their own account. Admin cannot delete their own account.
DELETE /users/:id Hard delete a user Admin only ▼
Response 200
{ "message": "User deleted successfully" }
